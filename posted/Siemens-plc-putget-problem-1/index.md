# 西门子 S7-1500 1200 PUT/GET 通信错误问题一则

今天在调试项目的时候，使用 S7-1200 与 S7-1500 S7 单边通信（PUT/GET）。测试了很长时间，一直没法成功通讯。1500 侧 put 数据，状态是error，状态码是 `STATUS = 16#0001`。1200 侧 put 数据也不提示错误，也没有状态码，数据也没有 put 成功，配置 put 数据块还是两个绿色的。

之前也遇到过这个问题，只是没有搜索相关文章和总结这个问题，今天记录一下。

---

## 一、故障现象：STATUS = 16#0001 与 NULL 指针报错

在本项目中，我使用 **S7-1500 (固件 V2.8)** 向 **S7-1200 (固件 V4.4)** 写入数据（PUT）。程序执行时出现 **通信错误**：PUT 块输出 `STATUS = 16#0001`。

### 1.1 错误码 0001 的本质

查看西门子博途帮助文档，16#0001)的含义非常明确：
> **通信未就绪 / 连接未建立。**

根据搜索，常见原因包括：

- 1200 侧未勾选 **“允许来自远程伙伴的 PUT/GET 通信”**。
- 硬件组态（含 S7 连接）未下载到 PLC。
- REQ 触发过快或未轮询，导致连接被占用。

---

## 二、核心原因：PUT/GET 指令版本与 PLC 固件的匹配性

在排除了程序逻辑错误后，我将 PUT 指令从 **V1.3** 降级为 **V1.1**，结果通信立即恢复正常。
这说明，在使用 putget 命令的时候，如果如何也连接不上，读写不了数据，可以尝试降低库的版本，或许可以解决问题。

经过搜索，发现西门子的 PUTGET 库和 PLC 固件版本有支持关系，也就是库的高版本需要 PLC 的高版本固件才能运行。

以下是官方及现场验证后的版本对照表：

| PUT/GET 版本 | 对应 TIA Portal | 1500 最低固件 | 1200 最低固件 | 关键特性 |
| :--- | :--- | :--- | :--- | :--- |
| **V1.1** | V13 ~ V14 SP1及以上 | **V2.0+** | **V4.0+** | 经典老版本，管脚仅支持 ANY/POINTER，兼容性最好。 |
| **V1.2** | V15 ~ V16及以上 | **V2.5+** | **V4.2+** | 1200 作为客户端主动通信正式稳定，支持自动建连。 |
| **V1.3** | V17 ~ V21及以上 | **V2.9+** | **V4.5+** | 错误码表细化，支持新固件特性。 |

### 2.1 故障定论

在这个项目中，我的硬件设备的配置是：

- **1500 固件 V2.8**
- **1200 固件 V4.4**
- **PUT 版本 V1.3** (由高版本 TIA 自动生成)

对照上表可知：

- V1.3 要求 1500 ≥ V2.9，而用户是 V2.8（**不满足**）。
- V1.3 要求 1200 ≥ V4.5，而用户是 V4.4（**不满足**）。

**结果**：在 TIA V20+ 中新建项目时，PUT/GET 默认被设置为 V1.3。由于 PLC 固件未达到要求，指令处于“超规格运行”状态，导致通信静默失败。降级到 V1.1 或 V1.2 后，指令与固件匹配，通信随即恢复。

### 2.2 最佳实践建议

针对 **1500 V2.8 + 1200 V4.4** 这一广泛存在的组合：

- ✅ **推荐使用 PUT/GET V1.2**（V1.1 过旧，V1.2 在 V2.8 上运行极其稳定）。
- ⚠️ **不建议**为了使用 V1.3 而去升级固件。

我则是暴力的降级到 V1.1 解决问题。因为现场调试，没有继续测试 V1.2 能否正常。按照搜索到的资料，使用 V1.2 也是能正常的，下次使用的时候再测试这个版本。

---

## 三、进阶技巧：PUT/GET 中 BOOL 变量的 ANY 指针寻址

### bool 变量的传输问题

如果直接传送 bool 变量，因为库的原因，一次 putget 只能传输 4 个 bool 值。如果有更多变量就需要多加几条 putget 指令，我通常会选择打包发送，将 1500 和 1200 侧的 8 个 BOOL 映射到一个 BYTE 的不同位上，然后使用 BYTE 进行传输：

```plaintext
ADDR_1 := P#DB99.DBX0.0 BYTE 1   // 1200 侧目标地址
SD_1   := P#DB10.DBX0.0 BYTE 1   // 1500 侧源地址
```

但是这种方式只减少 putget 语句，并不减少映射语句/命令。

---

## 四、总结

针对 S7-1500 与 S7-1200 的 PUT/GET 通信，建议遵循以下标准化流程：

1. **版本核对**：

    - 确认 1500/1200 固件版本。
    - 根据对照表选择匹配的 PUT/GET 指令版本（V1.2 是目前大多数现场的最优解）。

2. **连接配置**：
    - 1200 侧勾选“允许远程 PUT/GET”。
    - 网络视图中激活 S7 连接。
    - **务必下载硬件组态**。

3. **程序规范**：
    - 避免使用未赋值的 VARIANT 或 DB_ANY。
    - REQ 使用脉冲信号，多连接轮询。
    - 传输 BOOL 时，优先使用 BYTE 打包，且确保 DB 已取消优化访问。

**最终结论**：本项目中，通信失败的根本原因是 **TIA V20 生成的 PUT V1.3 指令与 1500 V2.8 / 1200 V4.4 固件不兼容**。降级指令版本并规范 ANY 指针写法后，问题彻底解决。

希望这篇文章能帮助你在面对类似西门子通信难题时，快速定位根因，少走弯路。
